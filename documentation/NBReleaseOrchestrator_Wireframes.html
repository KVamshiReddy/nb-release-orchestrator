Approve/Reject Panel

POST /api/v1/approvals/{id} with {decision:"APPROVED", reason:""}. Triggers Spring @EventListener → NotificationService → email + WebSocket push.

4 Tabs Architecture

Overview = release entity. Approvals = approvals table. Deployment = runbook fields. Audit = /audit endpoint (audit_logs table). Tabs visible by role.

Request Changes Flow

Release Mgr can add a comment that syncs back to Jira via JiraService.addComment(). Status stays PENDING_REVIEW. Dev team sees comment in both Jira and portal.

Initiate Deployment

POST /api/v1/releases/{id}/initiate — validates status=APPROVED, transitions to IN_PROGRESS, publishes Spring ApplicationEvent → all stakeholders notified via WebSocket + email.

Live Progress Updates

DevOps manually updates deployment step progress via PATCH /api/v1/releases/{id}/status. WebSocket pushes progress % to all connected dashboards in real-time.

Rollback Guard

Rollback button only active during IN_PROGRESS state. Confirmation modal requires a rollback reason (mandatory). Reason stored in audit_logs.notes field.

INSERT-Only Table

audit_logs is never UPDATEd or DELETEd. Written exclusively by AuditService @EventListener. No JPA update operations permitted on this entity.

Dual Source Tracking

Every log entry records SOURCE: PORTAL (action taken in NBReleaseOrchestrator) or JIRA_WEBHOOK (change came from Jira). Enables full cross-system traceability.

Compliance

Full audit trail supports SOX, PCI-DSS, and ISO 27001 change management requirements. Old/new state stored as JSONB for complete before/after diff.

Two-Channel Delivery

Spring ApplicationEvent → @Async @EventListener NotificationService → (1) JavaMailSender SMTP email, (2) SimpMessagingTemplate STOMP push to /user/queue/notifications.

Role-Filtered Recipients

NotificationService queries users by role to build recipient list per event type. DevOps gets deploy events. Stakeholders get high-level summary emails only.

Delivery Logging

Every notification attempt recorded in notifications table (delivered BOOLEAN, error_message if failed). Provides deliverability audit trail for compliance.
